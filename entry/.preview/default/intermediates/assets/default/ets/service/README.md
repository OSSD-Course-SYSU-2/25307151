# 语音识别与音频播放功能说明

## 功能概述

本应用已集成语音识别和自动音频播放功能：

1. **语音识别触发**：当检测到特定词汇"释怀"时，会自动播放指定的音频文件
2. **转盘旋转完成自动播放**：当转盘旋转完成后，会自动播放音频文件

## 使用前准备

### 1. 准备音频文件

**重要提示：由于版权保护，我无法提供三国OST的关羽之歌。你需要自己准备音频文件。**

请按照以下步骤操作：

1. 准备你想要播放的音频文件（支持 mp3、m4a 等格式）
2. 将音频文件重命名为 `trigger_sound.mp3`（或修改代码中的文件名）
3. 将音频文件放置到以下目录：
   ```
   entry/src/main/resources/rawfile/trigger_sound.mp3
   ```

### 2. 如果 rawfile 目录不存在

如果 `rawfile` 目录不存在，请手动创建：
```
entry/src/main/resources/rawfile/
```

## 使用方法

### 转盘旋转完成自动播放

转盘旋转完成后会自动播放音频文件：
1. 点击转盘开始旋转
2. 等待转盘停止
3. 自动播放音频文件

### 启动语音识别

1. 点击"开始监听"按钮启动语音识别
2. 状态会显示为"正在监听中"
3. 对着设备麦克风说出"释怀"
4. 系统检测到触发词后会自动播放音频

### 停止语音识别

- 点击"停止监听"按钮停止语音识别

### 测试播放

- 点击"测试播放"按钮可以测试音频文件是否正确放置

## 权限说明

应用已添加以下权限：
- `ohos.permission.MICROPHONE` - 用于语音识别的麦克风权限

首次使用时会请求麦克风权限，请允许授权。

## 自定义设置

### 修改触发词

在 `VoiceRecognitionService.ets` 中修改：
```typescript
private triggerWord: string = '释怀'; // 修改为你想要的触发词
```

### 修改音频文件名

在 `Index.ets` 中修改：
```typescript
await this.audioService.playFromRawFile('你的音频文件名.mp3');
```

## 技术实现

### 语音识别
- 使用 HarmonyOS 原生智能语音识别 SDK (`@kit.CoreSpeechKit`)
- 支持中文语音识别
- 实时检测识别结果中的触发词

### 音频播放
- 使用 HarmonyOS AVPlayer (`@ohos.multimedia.media`)
- 支持从 rawfile 资源播放
- 支持从应用沙箱路径播放

## 注意事项

1. **音频文件必须存在**：如果没有放置音频文件，播放会失败
2. **麦克风权限**：需要用户授权麦克风权限
3. **网络连接**：语音识别需要网络连接（使用在线识别模式）
4. **识别准确性**：语音识别可能受环境噪音影响，建议在安静环境使用

## 故障排除

### 无法启动语音识别
- 检查是否授予了麦克风权限
- 检查网络连接是否正常

### 无法播放音频
- 检查音频文件是否正确放置在 rawfile 目录
- 检查音频文件格式是否支持
- 查看日志输出的错误信息

### 识别不到触发词
- 确保发音清晰
- 尝试在安静环境使用
- 检查触发词设置是否正确
